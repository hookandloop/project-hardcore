Multiselect Test: Remove OptGroup Tags On All Selected.

Related JIRA Ticket. When selecting all child options of an optgroup in a Multiselect with the option moveSelectedToTop set to true (which is default), the empty optgroup should no longer be rendered by the Multiselect. Once one of its options becomes deselected, it should be re-rendered the next time the list opens.

By selecting all of the options in either the "fruits" or "vegetables" sections of the below Multiselect, the corresponding group header will longer render until one of those options once again becomes deselected.
